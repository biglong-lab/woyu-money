# 房客個資外洩通知書（草稿）

> 用途：依個資法 §12 主動通知 5 位房客個資已外洩
> 狀態：**草稿、未寄送**、由用戶決定是否使用
> 來源事件：[2026-05-31 secrets and backup hardening](2026-05-31-secrets-and-backup-hardening.md)

## 法律依據

個人資料保護法第 12 條：
> 公務機關或非公務機關違反本法規定，致個人資料被竊取、洩漏、竄改或其他侵害者，應查明後以適當方式通知當事人。

「適當方式」實務上接受：書面、email、簡訊、電話、LINE 訊息。
**時限**：發現後應「儘速」通知（無明確日數、建議 7 日內）。

## 通知名單（5 位）

從 `rental_contracts` 表查（已不公開）：

```sql
SELECT contract_name, tenant_name, tenant_phone
FROM rental_contracts
WHERE is_active = TRUE OR end_date >= '2026-02-04';  -- 暴露期間有效合約
```

## 寄送方式建議

1. **LINE 訊息**（最快、有讀取回執）
2. 紙本郵寄（最正式、保留寄達證明）
3. Email（如有 email）

## 風險評估

不通知的法律風險：
- 個資法 §47：對自然人外洩、若主管機關發現可處 5-50 萬罰鍰
- 民事：當事人可主張慰撫金（每人 500-20,000 NT、視情節）
- 影響商譽

建議通知。
